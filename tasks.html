Homework 1: Tasks for srs37

Your personal task is to develop a system for warehouses and their products.

Task 1: Configuration [4 marks]

Configure the project to support the use of JSPs as view technology. JSPs must be configured to be placed inside folder src/main/webapp/WEB-INF/views/.

Task 2: Domain Classes [13 marks]

Create class co2103.hw1.domain.Product. With the following private fields:

  1. String name
  2. String slogan
  3. String flavour
  4. int price
Generate necessary getters and setters. [7 marks]

Create class co2103.hw1.domain.Warehouse. With the following private fields:

  1. int id
  2. String founder
  3. String industry
  4. List<Product> products
Generate necessary getters and setters. [6 marks]

Task 3: Controller (simple) [5 marks]

Create controller co2103.hw1.controller.MainController [2 marks].

  1. Requests for URL "/" (POST and GET) should show the view start (the corresponding JSP already exists). [3 marks]

Task 4: Controller, View, and Form [20 marks]

Create field public static List<Warehouse> warehouses in class co2103.hw1.Hw1Application. Initialize the list inside class co2103.hw1.Hw1Application to contain one warehouse with two products. [4 marks]

Create controller co2103.hw1.controller.WarehouseController and add methods with request mappings for the following behavior:

  1. GET requests for URL "/warehouses" should show view warehouses/list. [2 marks]
    • Add the list of all warehouses from co2103.hw1.Hw1Application to the model as attribute warehouses. [2 marks]
    • Create the corresponding JSP to list the founder and industry of each warehouse followed by a link to URL "/products?warehouse={id of warehouse}", e.g., "/products?warehouse=3". [4 marks]
  2. Requests for URL "/newWarehouse" should show view warehouses/form. [3 marks] Create the corresponding JSP with a form with modelAttribute="warehouse" and action="/addWarehouse" and inputs for all fields of class Warehouse except for products. [5 marks]
  3. POST requests for URL "/addWarehouse" should add a new warehouse to the list of all warehouses in co2103.hw1.Hw1Application and redirect to /warehouses. [3 marks]

Task 5: Controller, View, and Form with additional Parameter [33 marks]

Create controller co2103.hw1.controller.ProductController and add methods with request mappings for the following behavior:

  1. GET requests for URL "/products" should show view products/list. [2 marks]
    • Based on request parameter warehouse determine the corresponding warehouse from co2103.hw1.Hw1Application. [2 marks]
    • Add the list of all products of that warehouse to the model as attribute products. [2 marks]
    • Create the corresponding JSP for the view to list the name, slogan, flavour, and price of each product. [4 marks]
    • At the end of the list in the JSP add a link to URL "/newProduct?warehouse={id of warehouse whose products are listed}", e.g., "/newProduct?warehouse=3". [4 marks]
  2. Requests for URL "/newProduct" should show view products/form. [3 marks]
    • Create the corresponding JSP with a form with modelAttribute="product" and action="/addProduct" and inputs for all fields of class Product. [6 marks]
    • Also add the request parameter warehouse with the ID of the current warehouse. [4 marks]
  3. POST requests for URL "/addProduct" should add a new product to the list of products of the warehouse identified by request parameter warehouse and then redirect to /warehouses. [6 marks]

Task 6: Validation [25 marks]

Set up a validator for class Warehouse and make it available in controller WarehouseController. [4 marks]

When adding a new warehouse validate that

  1. The value of field id is unique within the list Hw1Application.warehouses. [3 marks]
  2. The value of field founder is not empty. [2 marks]
  3. The value of field industry is not empty. [2 marks]
In case of validation errors go back to the form and show error messages for each field in the form.

Set up a validator for class Product and make it available in controller ProductController. [4 marks]

When adding a new product validate that

  1. The value of field name is not empty. [2 marks]
  2. The value of field slogan is not empty. [2 marks]
  3. The value of field flavour is either "Orange", "Banana", or "Apple". [3 marks]
  4. The value of field price is between (including) 1 and 3. [3 marks]
In case of validation errors go back to the form and show error messages for each field in the form.
